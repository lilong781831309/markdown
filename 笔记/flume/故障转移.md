#### 功能

监听端口数据,打印到logger1或logger2，logger1和logger2实现故障转移，logger2优先级高

logger2正常时，所有数据给logger2，

logger2挂掉时，数据给logger1

logger2恢复后,  所有数据重新给到logger2



#### 配置

netcat-memory-avro.conf

```shell
# Name
a1.sources = r1
a1.sinks = k1 k2
a1.channels = c1

# Source
a1.sources.r1.type = netcat
a1.sources.r1.bind = localhost
a1.sources.r1.port = 6666
a1.sources.r1.selector.type = replicating

# Sink
# sink1
a1.sinks.k1.type = avro
a1.sinks.k1.hostname = hadoop10
a1.sinks.k1.port = 8888
# sink2
a1.sinks.k2.type = avro
a1.sinks.k2.hostname = hadoop10
a1.sinks.k2.port = 9999

# sink processor
a1.sinkgroups = g1
a1.sinkgroups.g1.sinks = k1 k2
# default, failover or load_balance
a1.sinkgroups.g1.processor.type = failover
a1.sinkgroups.g1.processor.priority.k1 = 5
a1.sinkgroups.g1.processor.priority.k2 = 10
a1.sinkgroups.g1.processor.maxpenalty = 10000

# Channel
a1.channels.c1.type = memory
a1.channels.c1.capacity = 1000
a1.channels.c1.transactionCapacity = 100

# Bind
a1.sources.r1.channels = c1
a1.sinks.k1.channel = c1
a1.sinks.k2.channel = c1
```

#### 启动

```shell
flume-ng agent --conf $FLUME_HOME/conf --conf-file $FLUME_HOME/conf-file/failover/avro-memory-logger1.conf --name a1 -Dflume.root.logger=INFO,console

flume-ng agent --conf $FLUME_HOME/conf --conf-file $FLUME_HOME/conf-file/failover/avro-memory-logger2.conf --name a1 -Dflume.root.logger=INFO,console

flume-ng agent --conf $FLUME_HOME/conf --conf-file $FLUME_HOME/conf-file/failover/netcat-memory-avro.conf --name a1 -Dflume.root.logger=INFO,console
```
